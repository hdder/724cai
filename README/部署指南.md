# 服务器部署指南

## 一、一键部署（推荐）

### 使用自动部署脚本

项目提供了 `deploy_to_server.sh` 脚本，可以自动完成以下操作：

1. ✅ 扫描本地代码文件（排除 venv、node_modules、logs）
2. ✅ MD5 对比本地和服务器文件
3. ✅ 仅上传有改动的文件
4. ✅ 检测服务器上的多余文件并提示删除
5. ✅ 自动停止服务
6. ✅ 自动启动服务
7. ✅ 检查服务状态

### 配置部署脚本

编辑 `deploy_to_server.sh`，修改服务器信息：

```bash
SERVER_HOST="root@111.228.8.17"     # 服务器地址
SERVER_PASSWORD="1998113Yd2022"      # 服务器密码
SERVER_PATH="/home/724caixun"        # 服务器上的路径
LOCAL_PATH="/Users/hesiyuan/Code/724caixun"  # 本地路径
```

### 执行部署

```bash
chmod +x deploy_to_server.sh
./deploy_to_server.sh
```

### 安装 sshpass（Mac）

Mac 用户需要安装 sshpass 以实现自动登录：

```bash
brew install hudochenkov/sshpass/sshpass
```

如果不安装 sshpass，部署时需要手动输入密码。

## 三、Nginx 反向代理（可选）

### 何时使用 Nginx？

| 场景 | 是否需要 Nginx |
|------|----------------|
| 内部测试 | ❌ 不需要 |
| 初期部署 | ❌ 不需要 |
| 生产环境 | ✅ 推荐 |
| 需要 HTTPS | ✅ 必须 |
| 域名访问 | ✅ 推荐 |
| 高并发 | ✅ 必须 |

### 配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端页面
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
    }

    # Flask API
    location /api/ {
        proxy_pass http://127.0.0.1:5555;
        proxy_set_header Host $host;
    }

    # 外部接口
    location /admin/ {
        proxy_pass http://127.0.0.1:5555;
        proxy_set_header Host $host;
    }

    # WebSocket
    location /socket.io/ {
        proxy_pass http://127.0.0.1:9080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

## 五、防火墙配置

### 云服务器安全组规则

| 协议 | 端口 | 说明 | 是否必须 |
|------|------|------|----------|
| TCP | 22 | SSH | ✅ 必须 |
| TCP | 8000 | 前端 | 直接部署需要 |
| TCP | 9080 | WebSocket | 直接部署需要 |
| TCP | 5555 | API | 直接部署需要 |
| TCP | 80 | HTTP | 使用 Nginx 需要 |
| TCP | 443 | HTTPS | 使用 Nginx 需要 |

**使用 Nginx 后：** 只需开放 22、80、443

## 六、常见问题

### Q1: 端口被占用？

```bash
# 查看占用端口的进程
sudo lsof -i :8000

# 杀死进程
sudo kill -9 <PID>
```

### Q2: 防火墙阻止连接？

```bash
# 检查防火墙状态
sudo ufw status

# 开放端口
sudo ufw allow 8000
sudo ufw allow 9080
sudo ufw allow 5555
```

### Q3: WebSocket 连接失败？
- 检查 9080 端口是否开放
- 检查 Node.js 服务是否运行
- 查看日志：`tail -f logs/nodejs.log`

### Q4: 部署后访问不了？
1. 检查服务是否启动：`lsof -i:8000`
2. 检查防火墙是否开放端口
3. 查看日志：`tail -f logs/flask.log`
4. 强制刷新浏览器（Ctrl+Shift+R）

## 七、生产环境优化

### 1. 使用 PM2 管理 Node.js

```bash
sudo npm install -g pm2
cd websocket
pm2 start server.js --name 724caixun-push
pm2 startup
pm2 save
```

### 2. 使用 Gunicorn 运行 Flask

```bash
pip install gunicorn
cd backend
gunicorn -w 4 -b 127.0.0.1:5555 push_api:app
```

### 3. 配置 Systemd 服务

创建服务文件 `/etc/systemd/system/724caixun.service`：

```ini
[Unit]
Description=724caixun Push Service
After=network.target

[Service]
Type=simple
User=your-user
WorkingDirectory=/home/user/724caixun
ExecStart=/home/user/724caixun/start.sh
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start 724caixun
sudo systemctl enable 724caixun
```
