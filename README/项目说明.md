# 724财讯消息推送系统

实时消息推送平台，支持多频道管理和 Markdown 富文本消息。

## 系统简介

一个基于 **Flask + Socket.IO** 的实时消息推送系统：
- ✅ 支持多频道管理
- ✅ 实时 WebSocket 推送
- ✅ Markdown 消息格式
- ✅ 消息历史记录
- ✅ 简单的管理后台
- ✅ REST API 接口

## 快速开始

### 本地运行

```bash
# 克隆项目
cd 724caixun

# 启动所有服务
./start.sh
```

启动成功后：

- **用户页面**: http://localhost:8000
- **管理后台**: http://localhost:8000/admin.html?admin_token=724caixun_admin_2024_k9HxM7qL
- **API 接口**: POST http://localhost:5555/push/send?channel_token=xxx

详细页面说明: [页面说明.md](页面说明.md)
查看 API 文档: [API接口文档.md](API接口文档.md)

## 系统架构

```
前端 (Port 8000)  →  Flask API (Port 5555)  →  SQLite 数据库
                       ↓
                 Node.js (Port 9080)  →  WebSocket 实时推送
```

- **Flask**: 业务逻辑中心，负责数据库、API、会话管理
- **Node.js**: 纯 WebSocket 传输层，无状态
- **前端**: 静态 HTML 页面

详细架构说明: [ARCHITECTURE.md](ARCHITECTURE.md)

## 核心功能

### 1. 频道管理
- 创建、编辑、删除频道
- 自动生成频道 ID (1, 2, 3...)
- 自动生成推送令牌 (Token)

### 2. 消息推送
- 支持 Markdown 格式
- 图片、链接、列表等富文本
- 实时推送到所有在线用户
- 消息持久化存储

### 3. 用户访问
- 无需登录，通过 Token 访问
- WebSocket 实时连接
- 自动重连机制

### 4. REST API
```bash
# 发送消息
curl -X POST "http://localhost:5555/push/send?channel_token=YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "msgtype": "markdown",
    "markdown": {
      "title": "消息标题",
      "text": "消息内容"
    }
  }'
```

## 管理后台使用

### 访问管理后台

```
http://your-server:8000/admin.html?admin_token=724caixun_admin_2024_k9HxM7qL
```

### 功能说明

1. **添加频道**
   - 点击"添加频道"按钮
   - 输入频道名称（如：新闻资讯）
   - 系统自动生成频道 ID 和 Token
   - 复制 Token 供 API 调用使用

2. **编辑频道**
   - 点击频道行的"编辑"按钮
   - 修改频道名称或 Token
   - 点击"保存"

3. **删除频道**
   - 点击频道行的"删除"按钮
   - 确认后删除整个频道及其所有消息
   - ⚠️ 此操作不可恢复

4. **复制 Token**
   - 点击 Token 列的复制按钮
   - Token 自动复制到剪贴板

## 开发说明

### 数据库位置

```
websocket/data/push_messages.db
```

### 主要 API 端点

- `POST /push/send?channel_token=xxx` - 发送消息
- `GET /api/channels` - 获取频道列表
- `GET/POST /api/tokens` - Token 管理
- `GET /health` - 健康检查

### WebSocket 事件

- `subscribe` - 订阅频道
- `switch_channel` - 切换频道
- `new_message` - 接收新消息

## 常见问题

**Q: WebSocket 连接失败？**
A: 检查 9080 端口是否开放，Node.js 服务是否运行

**Q: 消息发送失败？**
A: 检查 channel_token 是否正确，Flask 服务是否运行

**Q: 管理后台无法访问？**
A: 确认 admin_token 参数正确

**Q: 如何查看服务日志？**
A: 查看 logs/ 目录下的日志文件
