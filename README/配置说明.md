# 配置管理说明

## 概述

项目已统一管理所有IP地址和端口号，不再硬编码在代码中。

## 配置文件

### 主配置文件：`config.json`

位于项目根目录，包含所有服务器的配置信息：

```json
{
  "server": {
    "host": "0.0.0.0",
    "public_ip": "111.228.8.17",
    "domain": ""
  },
  "ports": {
    "flask_backend": 5555,
    "websocket_server": 9080,
    "frontend": 8000
  },
  "urls": {
    "flask_backend": "http://111.228.8.17:5555",
    "websocket_server": "ws://111.228.8.17:9080",
    "frontend": "http://111.228.8.17:8000",
    "push_api": "http://111.228.8.17:5555/push/send"
  },
  "admin": {
    "token": "724caixun_admin_2024_k9HxM7qL",
    "url": "http://111.228.8.17:8000/admin.html"
  }
}
```

## 如何修改配置

### 1. 修改服务器IP地址

编辑 `config.json` 文件：

```json
{
  "server": {
    "public_ip": "你的新IP地址"
  }
}
```

### 2. 修改端口

```json
{
  "ports": {
    "flask_backend": 你的Flask端口,
    "websocket_server": 你的WebSocket端口,
    "frontend": 你的前端端口
  }
}
```

### 3. 配置自动生效

修改配置文件后，需要重启相关服务：

```bash
# 重启所有服务
cd /path/to/724caixun
./stop.sh
./start.sh
```

## 配置文件使用

### 后端 Python 代码

```python
from config import FLASK_PORT, FLASK_URL, PUBLIC_IP

# 使用配置
print(f"Flask运行在: {FLASK_URL}")
```

### WebSocket Node.js 代码

```javascript
const config = require('./config');

// 使用配置
const PORT = config.WEBSOCKET_PORT;
```

### 前端 JavaScript 代码

```javascript
// 自动从config.js加载
const API_BASE = getApiBase();
const WS_URL = getWebSocketUrl();
```

## 本地开发配置

本地开发时，前端会自动检测localhost，无需修改配置。

如需指定其他服务器，可在URL中添加参数：

```
http://localhost:8000/admin.html?config_host=192.168.1.100&config_port=8000
```

## 配置文件结构

- **config.json** - 主配置文件（项目根目录）
- **backend/config.py** - Python配置加载器
- **frontend/config.js** - 前端配置加载器
- **websocket/config.js** - WebSocket配置加载器

## 注意事项

1. 修改配置后务必重启所有服务
2. 确保端口未被占用
3. 防火墙需要开放相应端口
4. 公网部署时修改 `public_ip` 为实际IP
5. 生产环境建议使用域名而非IP

## 默认端口

- Flask后端：5555
- WebSocket服务：9080
- 前端页面：8000
