```

       │
```

## 核心设计原则

**Flask 作为唯一的业务逻辑中心**
- 所有业务状态存储在数据库
- 所有订阅、未读数逻辑由 Flask 管理
- Node.js 仅作为无状态的 WebSocket 传输层

**Node.js 作为纯传输层**
- 不维护任何业务状态
- 不操作数据库
- 仅负责 WebSocket 消息的发送和接收

## 组件职责

### 1. Flask 后端 (`backend/`)

**职责：**
- HTTP API 服务
- 所有业务逻辑处理
- SQLite 数据库管理
- Token 认证和管理
- 消息存储和历史记录
- 用户会话管理
- 用户订阅管理
- 未读消息计数

**主要接口：**
- `POST /push/send?channel_token=xxx` - 外部程序发送消息
- `GET/POST /api/tokens` - Token 管理
- `GET /api/channels` - 获取频道列表
- `GET /health` - 健康检查
- `POST /api/ws/connect` - WebSocket 连接时创建会话
- `POST /api/ws/disconnect` - WebSocket 断开时删除会话
- `POST /api/ws/subscribe` - 订阅频道
- `POST /api/ws/switch-channel` - 切换当前频道

### 2. Node.js 推送服务 (`websocket/`)

**职责：**
- WebSocket 连接管理
- 消息实时推送到前端
- **不负责业务逻辑**
- **不维护状态**
- **不操作数据库**

**主要接口：**
- `POST /api/push` - 接收 Flask 的推送请求
- `GET /health` - 健康检查

**WebSocket 事件：**
- `connection` → 调用 Flask `POST /api/ws/connect`
- `subscribe` → 调用 Flask `POST /api/ws/subscribe`
- `switch_channel` → 调用 Flask `POST /api/ws/switch-channel`
- `disconnect` → 调用 Flask `POST /api/ws/disconnect`

### 3. 前端 (`frontend/`)

**页面：**
- `index.html` - 消息接收页面
- `admin.html` - 管理后台

**功能：**
- WebSocket 连接
- 消息实时显示
- 频道管理
- Token 管理

## 数据库结构

### channels 表（频道）
```sql
id TEXT PRIMARY KEY,
name TEXT NOT NULL,
description TEXT,
created_at TIMESTAMP
```

### messages 表（消息）
```sql
id INTEGER PRIMARY KEY,
channel_id TEXT NOT NULL,
title TEXT,
content TEXT NOT NULL,
created_at TIMESTAMP
```

### channel_tokens 表（令牌）
```sql
id INTEGER PRIMARY KEY,
channel_id TEXT NOT NULL,
token TEXT UNIQUE NOT NULL,
created_at TIMESTAMP
```

### user_sessions 表（用户会话）
```sql
socket_id TEXT PRIMARY KEY,
current_channel TEXT,
created_at TIMESTAMP,
last_active TIMESTAMP
```

### user_subscriptions 表（用户订阅）
```sql
id INTEGER PRIMARY KEY,
socket_id TEXT NOT NULL,
channel_id TEXT NOT NULL,
subscribed_at TIMESTAMP,
UNIQUE(socket_id, channel_id)
```

## 设计优势

### Flask 作为唯一业务逻辑中心

1. **状态一致性**
   - 所有业务状态在数据库中持久化
   - 不会因为 Node.js 重启丢失状态
   - 多个 Node.js 实例可以共享同一个 Flask

2. **逻辑集中**
   - 订阅管理、未读数计算在 Flask 统一处理
   - Node.js 无需理解业务逻辑
   - 便于维护和调试

3. **数据库生态**
   - SQLite3 内置支持
   - 查询未读数等复杂逻辑在 SQL 层面处理
   - 易于扩展到 PostgreSQL/MySQL

### Node.js 作为纯传输层

1. **无状态设计**
   - 不维护内存状态
   - 可以水平扩展
   - 重启不影响业务逻辑

2. **专注传输**
   - Socket.IO 成熟稳定
   - 长连接管理高效
   - 实时推送性能好

3. **简化职责**
   - 代码简洁（仅 130 行）
   - 不操作数据库
   - 易于维护和替换

## 部署说明

### 启动顺序：
1. Flask 后端（创建数据库表）
2. Node.js 推送服务
3. 前端 HTTP 服务

### 依赖关系：
- Node.js 依赖 Flask（通过 HTTP 调用）
- Flask 不依赖 Node.js（推送失败不影响业务逻辑）
- 前端依赖两者（WebSocket + HTTP API）

### 端口分配：
- 8000 - 前端页面
- 9080 - WebSocket 推送
- 5555 - Flask API

## 扩展性

### 水平扩展：
- Flask 可以多实例部署（负载均衡 + 共享数据库）
- Node.js Socket.IO 可以多实例部署（无需状态同步）
- 数据库可以升级到 PostgreSQL/MySQL

### 功能扩展：
- 添加用户认证（Flask）
- 消息审核机制（Flask）
- 推送统计（Flask 数据库查询）
- 消息加密（传输层）
